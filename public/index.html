Authentication app

This project represents a REST API for a chat application, providing user registration, authentication, session management, message sending, and interaction with a database. The API allows for creating, reading, updating, and deleting (CRUD) information related to users and messages. Protection against XSS attacks and password hashing ensure secure server interaction.

Main Features:

  • User Registration: Users can register by creating an account with a name, email, and password. The password is hashed using bcrypt.

  • Authentication and Authorization: Users can authenticate using JWT (JSON Web Token) for secure access to the service. Tokens are generated and verified using jsonwebtoken.

  • Chat and Messaging: Users can send and receive messages within the chat. This data is managed through the database, likely using Prisma.

  • Session Management: Users can log in, and their sessions are stored using cookies, allowing for easy user state management.

  • XSS Attack Protection: xss-clean is used to protect against malicious script injection in input fields.
